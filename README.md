## コンセプト
時間の凍土に埋もれた記憶を、見知らぬ誰かと掘り起こし、温め合う。

## 対象ユーザ
利用の流れ：
アプリを起動後、ホームセクションに移る。このセクションでは、この時点ではユーザはゲストとしてこのアプリを利用している。ホームの右上
にあるGと書かれているアイコンからログイン/カウントの新規登録をする。自身のアカウントでログイン後、思い出となる写真を投稿することが
できる。ホームセクションを左にスワイプすると、思い出の発掘画面に移る。この画面では、ユーザ自身が投稿した思い出の写真を発掘するので
はなく、ほかのユーザが投稿した思い出の写真を発掘する。発掘したほかのユーザの投稿には”いいね”を押すことができる。発掘画面を左にス
ワイプすると、実績画面に移る。ユーザが発掘した回数、いいねした回数、ユーザ自身の投稿が発掘された回数、ユーザがいいねされた回数の４
つの項目に対して実績が課される。それぞれの項目には、回数に応じて４段階の称号が用意されている。ユーザはこれらの３つのセクションでア
プリを楽しめる。

##推しポイント：
- 自分の投稿を自分で見れないシステムにした。つまり、自分の投稿は自分では発掘できず、ホーム画面にも並ばない。この意図としては、自分
　の思い出を見知らぬ誰かへのプレゼントとしようということである。
- 一瞬のスクロールで写真を写さず、何度も写真をタップして凍土の中から発掘するという苦労を介することで、一つの写真に対する心理的な距
　離が縮まり、いいねやコメントによる深い共感が生まれるように設計した。
- 誰が投稿したのかは、発掘してからでないとわからないようにした。これは、だれが投稿したのかではなく、その投稿にある記憶、思い出はど
　ういうものなのかをフォーカスさせることで、過剰なコミュニケーションをなくし、純粋ないいねやコメントが生まれるように設計した。

### 開発における工夫した点
私たちのチームは開発期間にもかかわらず、皆がバイトやサークルの用事でまとまった時間が取れなかったが、
授業や課題をおろそかにせず、メリハリつけるためにも、バイトやサークルも開発のために休みにさせることなく開発を行った。
それを実現するためにも基本的にリモートでの開発を基本としたが
チームメイトが対面で話し合える時間があればできるだけ対面で開発するように努めた(特にテスター時)。
リモートでの開発の際、Discordを活用して毎日夜に進捗確報告を各タスクの担当から行い、それを踏まえて明日以降の流れを確認して１日を終えるミーティングを行い、PMが中心となって各メンバーへのタスク管理をNotionで行った。
技術面では、短期間での開発のため、初日～1日目にFigmaでアイデア出しを行い、プロトタイプを同じくFigmaで作成し、アプリの方向性を決めた。方向性がきまったら、FigmaAIでかいてくれたプロトタイプをCursorを用いてFlutterに変換し後は変換後のFlutter環境でCursor標準搭載のAIコードエージェントと対話しながら(バイブコーディング)コードを書いた。
AIが全面に書いたコードではあるが、ある程度動くようになった段階で人間にもテストし、UI,UXの改善案を思考発話法で。まとめた。
また、AIが書いたコードは過去の情報から拾ってくるため、最新の脆弱性に対応できていない場合があることを懸念して、MobSUBによるセキュリティ診断、ペネトレーションテストを行い、開発して終わりの政策ではなく、長く使ってもらえるような開発を行った。
