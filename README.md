## コンセプト
時間の凍土に埋もれた記憶を、見知らぬ誰かと掘り起こし、温め合う。

## 対象ユーザ
このプロダクトは「リアルタイム」に疲弊し、SNSを「義務」のように感じている人、スマホ内にはたくさんの写真が残っているが、見返すことが
ほとんどない人を対象に開発した。私たちのチームはこれらの人たちが抱えている問題について、SNSに追われて心が休まらないこと、他人と比較
して自己肯定感が下がることだろうと考えた。例えば、SNS通知が来ると反射的に開いてしまったり、空いた時間に無意識にタイムラインを見てし
まうことを行動の特徴として考え、SNS疲れから距離を取り、心を休める時間を提供しようとプロダクト開発を進めた。


## 利用の流れ
アプリを起動後、「探索を始める」をタップする。初回起動時ではユーザはサインインを求められるので、「アカウントをお持ちでない方は新規
登録」をタップし、ユーザのメールアドレス、パスワードを設定し、ホームセクションに移る。このセクションでは、ユーザが発掘したほかのユ
ーザの投稿が閲覧できる。画面右下の＋マークをタップすると思い出を投稿できる。投稿の際に題名と思い出のキラキラ度を設定できる。ホーム
セクションを右にスワイプすると、思い出の発掘セクションに移る。このセクションでは、ユーザ自身が投稿した思い出の写真を発掘するのでは
なく、ほかのユーザが投稿した思い出の写真を発掘する。発掘したほかのユーザの投稿にはコメントとキラキラを送ることができる。ホームセク
ションを左にスワイプすると、実績セクションに移る。ユーザが発掘した回数、いいねした回数、ユーザ自身の投稿が発掘された回数、ユーザが
いいねされた回数の４つの項目に対して実績が課される。それぞれの項目には、回数に応じて４段階の称号が用意されている。ユーザはこれらの
３つのセクションでアプリを楽しめる。

## 推しポイント
- 発掘したほかのユーザの写真をコレクションとしてホームに表示される。
- 一瞬のスクロールで写真を写さず、何度も写真をタップして凍土の中から発掘するという苦労を介することで、一つの写真に対する心理的な距
離が縮まり、いいねやコメントによる深い共感が生まれるように設計した。
- ユーザが長く使ってくれるようモチベーション維持のために、実績と称号機能を追加した

### 開発における工夫した点
私たちのチームは開発期間にもかかわらず、皆がバイトやサークルの用事でまとまった時間が取れなかったが、
授業や課題をおろそかにせず、メリハリつけるためにも、バイトやサークルも開発のために休みにさせることなく開発を行った。
それを実現するためにも基本的にリモートでの開発を基本としたが
チームメイトが対面で話し合える時間があればできるだけ対面で開発するように努めた(特にテスター時)。
リモートでの開発の際、PMが中心となって各メンバーへのタスク管理をNotionでし、Discordを活用して毎日夜に１日の進捗確認報告を各タスクの担当から行った。それらを踏まえて明日以降の流れをNotionを用いてPMが各メンバーへ割りふり、各メンバーへやることをDiscordに通知させた。
技術面では、短期間での開発のため、初日～1日目にFigmaでアイデア出しを行い、プロトタイプを同じくFigmaで作成し、アプリの方向性を決めた。方向性がきまったら、FigmaAIでかいてくれたプロトタイプをCursorを用いてFlutterに変換し後は変換後のFlutter環境でCursor標準搭載のAIコードエージェントと対話しながら(バイブコーディング)コードを書いた。
AIが全面に書いたコードではあるが、ある程度動くようになった段階で人間にもテストし、UI,UXの改善案を思考発話法で。まとめた。
また、AIが書いたコードは過去の情報から拾ってくるため、最新の脆弱性に対応できていない場合があることを懸念して、MobSFによるセキュリティ診断、ペネトレーションテストを行い、開発して終わりの政策ではなく、長く使ってもらえるような開発を行った。
